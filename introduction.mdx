FastMCP is a TypeScript framework for building [Model Context Protocol (MCP)](https://modelcontextprotocol.io) servers capable of handling client sessions. Built on top of the official MCP SDK, FastMCP eliminates complexity by providing an opinionated framework with simple, intuitive APIs.

## Why FastMCP?

The official MCP SDK provides foundational blocks for building servers, but leaves many implementation details to you:

- Initiating and configuring all server components
- Handling connections and sessions
- Managing tools, resources, and prompts
- Processing responses and errors
- Embedding resources, images, and audio content

FastMCP eliminates this complexity by:

- Handling all the boilerplate automatically
- Providing simple, intuitive APIs for common tasks
- Including built-in best practices and error handling
- Letting you focus on your server's core functionality

## When to use FastMCP

**Choose FastMCP when you want to:**
- Build MCP servers quickly without dealing with low-level implementation details
- Focus on your server's functionality rather than infrastructure
- Use best practices and error handling out of the box

**Use the official SDK when you:**
- Need maximum control over server architecture
- Have specific requirements that FastMCP doesn't address
- Want to reference FastMCP's implementation to avoid common pitfalls

## Architecture overview

FastMCP servers consist of three main components:

1. **Tools** - Executable functions that can be invoked by clients and LLMs to perform actions
2. **Resources** - Any kind of data you want to make available (files, images, logs, etc.)
3. **Prompts** - Reusable prompt templates and workflows for standardized LLM interactions

All components are defined with simple APIs and automatically exposed through the MCP protocol.
